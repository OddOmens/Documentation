# Statuses & Workflow

Track idea progress through workflow stages.

## Default Statuses

### 1. Unassigned (Gray)
- New ideas without status
- Starting point
- Needs categorization

### 2. Planning (Blue)
- Ideas in planning phase
- Research and development
- Defining scope

### 3. Started (Green)
- Active development
- Work in progress
- Currently executing

### 4. On Hold (Orange)
- Temporarily paused
- Waiting for resources
- Blocked by dependencies

### 5. Completed (Green)
- Finished ideas
- Successfully executed
- Achieved goals

### 6. Cancelled (Red)
- Abandoned ideas
- No longer pursuing
- Decided against

### 7. Archived (Purple)
- Stored for reference
- Historical record
- Not actively working

## Changing Status

### Method 1: Quick Swipe

**Swipe Right on Idea:**
- **Complete** - Mark as completed
- **Start** - Begin working
- **Hold** - Pause temporarily

### Method 2: Idea Details

1. Open idea
2. Tap Status section
3. Select new status from list
4. Updates immediately

### Method 3: Bulk Archive

1. Menu → Bulk Archive Ideas
2. Select multiple ideas
3. Tap archive icon
4. All selected ideas archived

## Status Visibility

Control which statuses appear in Dashboard.

### Settings → Features → Ideas

**Toggle Options:**
- **Show Cancelled Ideas**
- **Show Completed Ideas**
- **Show Archived Ideas**

**Benefits:**
- Cleaner Dashboard
- Focus on active work
- Hide finished ideas
- Reduce clutter

## Custom Statuses

### Settings → Statuses

**Customization Options:**
- Enable/disable default statuses

- Change status colors
- Reorder status display

**Use Cases:**
- Match your workflow
- Industry-specific terms

- Personal preferences

## Workflow Strategies

### Linear Workflow

**Simple Progression:**
1. Unassigned → Planning
2. Planning → Started
3. Started → Completed

**Best for:**
- Personal projects
- Simple ideas
- Quick execution

### Iterative Workflow

**Flexible Progression:**
1. Unassigned → Planning
2. Planning → Started
3. Started ↔ On Hold
4. Started → Completed

**Best for:**
- Complex projects
- Team collaboration
- Long-term ideas

### Review Workflow

**With Archive:**
1. Unassigned → Planning
2. Planning → Started
3. Started → Completed
4. Completed → Archived (after review)

**Best for:**
- Portfolio building
- Historical reference
- Learning from past

## Status Best Practices

### Regular Updates

- Update status as work progresses
- Weekly status review
- Keep Dashboard current
- Reflect actual state

### Clear Criteria

**Planning:**
- Research phase
- Defining requirements
- Creating tasks

**Started:**
- Active work
- Tasks in progress
- Regular updates

**On Hold:**
- Waiting for input
- Resource constraints
- Temporary pause

**Completed:**
- All tasks done
- Goals achieved
- Ready to archive

### Bulk Operations

**Bulk Archive:**
- End of quarter cleanup
- Project completion
- Seasonal review
- Keep Dashboard focused

**Bulk Status Change:**
- Use swipe actions
- Quick updates
- Efficient management
