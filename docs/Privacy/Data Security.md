# Data Security

How we protect your information across all Odd Omens apps.

## Security by Design

### Local-First Architecture

**All data stored on your device:**
- No remote servers to hack
- No databases to breach
- No cloud storage we control
- Your device, your data

**Benefits:**
- Complete control
- Offline functionality
- Fast performance
- Maximum privacy

### iOS Security Features

**Built on Apple's security:**
- **Encryption at rest** - Data encrypted on device
- **Secure enclave** - Hardware-level security
- **App sandboxing** - Isolated from other apps
- **Code signing** - Verified by Apple

## iCloud Security

### When You Enable iCloud Sync

**Apple's security protections:**
- **End-to-end encryption** - Data encrypted in transit
- **Encrypted storage** - Data encrypted in iCloud
- **Two-factor authentication** - Protect your Apple ID
- **Device authorization** - Control which devices sync

**We never have access:**
- Data goes directly to your iCloud
- We can't read your iCloud data
- We can't access your account
- Only you have the keys

### iCloud Best Practices

**Protect your iCloud:**
1. Enable two-factor authentication
2. Use strong Apple ID password
3. Don't share your Apple ID
4. Review authorized devices regularly
5. Keep iOS updated

**Encryption:**
- iOS encrypts all app data

- Device passcode required

## Network Security

### Minimal Network Usage

**Most apps work offline:**
- No constant internet connection needed
- Data doesn't leave your device
- Network only for iCloud sync
- No tracking servers

**When we use network:**
- iCloud sync (encrypted)
- App updates (signed by Apple)
- Crash reports (anonymous, optional)

### No Third-Party Servers

**We don't use:**
- Analytics servers
- Advertising networks
- Social media SDKs
- Tracking services
- External databases

## Permissions

### What We Request

**Notifications:**
- For reminders and alerts
- Optional, can be disabled
- No tracking via notifications

**Photos:**
- For adding images to ideas/thoughts
- For receipt scanning
- Access only when you select photos
- No automatic uploads

**Camera:**
- For receipt scanning (Fundful)
- For taking photos (Depthful, Ideaful)
- Only when you use camera feature

**Microphone:**
- For voice recordings (Depthful)
- Only when recording
- Processed on-device

**Speech Recognition:**
- For transcription (Depthful)
- Processed on-device
- No audio sent to servers

**HealthKit:**
- For health data sync (Breatheful, Afternoon)
- Optional integration
- You control what syncs

**Calendar:**
- Read-only access (Afternoon)
- View events only
- No modifications

**Screen Time:**
- For app blocking (Distant)
- Apple's official API
- No tracking of usage

### Permission Best Practices

**We follow Apple's guidelines:**
- Request only when needed
- Explain why we need it
- Respect your choice
- Work without permissions when possible

## Data Retention

### How Long We Keep Data

**On Your Device:**
- Forever, until you delete it
- You control retention
- Delete anytime

**In Your iCloud:**
- Until you delete it
- Managed by you
- Not by us

**Crash Reports:**
- Automatically deleted after 90 days
- Anonymous only
- Optional

**We Don't Retain:**
- Personal information (we don't collect it)
- Usage analytics (we don't track it)
- Deleted data (it's gone)
